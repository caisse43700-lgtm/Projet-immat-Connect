## BUGS FONCTIONNELS (impact utilisateur réel)

### BUG-001 — quickMsg() et quickReply() : champs morts après migration
- **Fichier** : index.html lignes 699, 790
- **Gravité** : Haute
- **Symptôme** : Le panneau Messages s'ouvre mais le compose est vide. Le message tapé dans la réponse rapide (`t`) est perdu.
- **Cause** : Ces fonctions écrivent dans `$('iMsg')` et `$('iTarget')` — champs de `panelContact` (caché). L'install() de compatibilité (ligne 1606) redirige `panel('contact')` → `panel('messages')`, mais ne transfère pas les données.
- **Contexte** : `vehicleAlert()` et `pickPlate()` ont déjà été migrés par l'install block. `quickMsg`/`quickReply` ne l'ont pas été.
- **Fix** : Réécrire ces 2 fonctions pour cibler `icComposePlate` / `icComposeText` + appeler `ImmatMessages.setMode('compose')`.
- **Statut** : ✅ CORRIGÉ dans ce commit

### BUG-002 — Debug tools visibles de tous les utilisateurs
- **Fichier** : index.html ligne 214 (panelSettings)
- **Gravité** : Haute (FRI-004, D-007)
- **Symptôme** : "📨 Restaurer msgs" et "🔄 Sync alertes" apparaissent dans les paramètres de tous les conducteurs. Confusion, sentiment d'interface cassée.
- **Cause** : Boutons non conditionnés au rôle Gardien.
- **Fix** : CSS `display:none` + classe `gardien-debug-tool`, révélés quand `S.isGardien = true`.
- **Statut** : ✅ CORRIGÉ dans ce commit

### BUG-003 — brain-dialog : pas de retry sur erreur RPC rôle
- **Fichier** : supabase/functions/immat-brain-dialog/index.ts ligne 220
- **Gravité** : Moyenne
- **Symptôme** : Si `get_my_role()` échoue en cold start (Supabase lent), le Gardien reçoit un 403 et doit ré-essayer manuellement.
- **Cause** : `Promise.all([auth, rpc])` — si rpc échoue, return 403 immédiat, aucun retry.
- **Fix** : 1 retry avec 600ms backoff si `roleErr`.
- **Statut** : ✅ CORRIGÉ dans ce commit

## DETTE TECHNIQUE (pas de bug, à surveiller)

### DET-001 — Références lignes index.html dans NS ✅ TRAITÉ (SESSION 11)
- `immat-nervous-system.json` documentait des entry points par numéro de ligne (ex: `"afterAuth": "index.html:507"`).
- **Analyse** : 13 dérives détectées (jusqu'à -521 lignes). Les anchors ligne étaient trop fragiles pour TRF-003.
- **Fix SESSION 11** : 19 entrées `index.html:LINE` remplacées par des anchors symboliques (`App.method`, `fn.func`, `db.table.op`, `css.selector`, `#id`). nsToPrompt() mis à jour. L'Ange voit désormais "afterAuth@App.afterAuth" au lieu de "afterAuth@507".
- **scripts/validate-ns-refs.js** : CI script vérifiant que chaque anchor existe dans la codebase. Tous les 21 anchors résolus.
- **Statut** : ✅ FAIT (SESSION 11)

### DET-002 — nsToPrompt() : pas de validation schéma NS au démarrage ✅ TRAITÉ (SESSION 11)
- Si NS change de structure (renommage d'un champ), `nsToPrompt()` échouait silencieusement (retournait un prompt incomplet) plutôt que de crasher avec une erreur claire.
- **Fix SESSION 11** : `validateNSSchema()` ajoutée dans `brain-dialog/index.ts`. Vérifie 5 champs obligatoires (`organs`, `routing`, `inhibitions`, `invariants`, `ange_identity`) avec messages d'erreur explicites par champ. Appelée avant `nsToPrompt(3)` au cold start.
- **Statut** : ✅ FAIT (SESSION 11)

### DET-003 — Cross-références UX architecture faibles
- UX-BUTTONS.json, UX-INTERACTIONS.json, UX-JOURNEYS.json, UX-FRICTIONS.json et UX-DECISIONS.md coexistent sans liens bidirectionnels entre leurs IDs (BTN-*, INT-*, JRN-*, FRI-*, D-*).
- **Recommandation** : Ajouter des champs `"frictions": ["FRI-001"]` dans les interactions, `"interactions": ["INT-002"]` dans les boutons, etc. Rend la navigation entre fichiers possible.
